## Phase 1 — Cheap correctness/security quick-wins (do alongside H02) · all S

These are low-effort, high-value, and several become *visible* the moment H02 lands
(the eval harness will show their quality impact).

- [x] **H03** · P0 · S · embeddings — **Embedding dim/model mismatch unvalidated → silent corruption.** ✅ COMPLETE (spec 005): `Vector.Query` skips mismatched-length vectors (no silent garbage cosine); `engine.Query` refuses on model/dim mismatch via `ErrEmbeddingMismatch`; status surfaces stored majority + drift. *(Audit §1.2)*
- [x] **H09** · P0 · S · retrieval — **Reranker errors silently swallowed.** On rerank error or length mismatch: log the error, surface a `RerankFailed` flag, optionally re-run with a larger pool; never discard the error. *(Audit §1.4)* → **spec 006** (implemented: `RerankFailed` flag on QueryResult surfaced across CLI/REST/gRPC/MCP; retrieval-error propagated (FR-009); opt-in retry (FR-006); failure logged w/o query text (FR-003). Gates: build/vet/test/eval green; `golangci-lint` skipped — not installed in env)
- [x] **H13** · P0 · S · security — **Loopback not enforced on bare `serve` (default binds `0.0.0.0`).** Make loopback the default in `config.Default()`; reject non-loopback bind at boot unless `--bind-external` is set. *(Audit §1.7/§1.8)* ✅ COMPLETE (spec 007): `internal/daemon/bind.go` (`IsLoopbackBind`/`ValidateBind`/`ExternalBindWarning`); boot gate in `serve`'s RunE (chokepoint) + fast-fail pre-check in `start`; `--bind-external` on `start`+`serve` forwarded via `Addrs.BindExternal`; exposure warning at boot; `config.Default().MCPAddr` → `127.0.0.1:7878` + regression test; README docs. - [x] **H07** · P1 · S · embeddings — **Missing embedding instruction-prefix (nomic/E5).** ✅ COMPLETE (spec 008): a pure `Prefixer` (`internal/embed/prefix.go`) applies role-correct `search_query:` (retrieval) / `search_document:` (pipeline) prefixes via a nomic/E5/BGE default map, config-gated (`embedding_prefix` auto|on|off + per-role overrides); applied at the two embed boundaries (pipeline documents, `engine.Query` query wrap) with `Embedder.Embed` unchanged (Principle V); the 0x04 record gains a `convention` provenance axis extending spec 005's `EmbeddingProfile` + mismatch guard (refuse on convention mismatch or mixed — never silently half-prefix, FR-006); status surfaces the active convention/mode; re-embed is no-duplicate (Principle II). Gates: build/vet/test green (`-race -cover`); real-nomic smoke verified (ingest+query+status with prefixes on `nomic-embed-text`, 768-dim); `golangci-lint` skipped — not installed in env. SC-001 measured via opt-in BEIR benchmarks (`go-rag eval --benchmark {scifact,msmarco}`, `internal/eval/beir`): on **SciFact** (low query/passage asymmetry) prefixes are near-neutral (recall@5 0.735→0.744, recall@10 0.813→0.807); on **MS MARCO** (high asymmetry, 200 dev queries + 8k distractor subsample) prefixes are **positive on every metric** — recall@5 0.988→0.998, recall@10 0.995→1.000, MRR 0.954→0.971, NDCG 0.964→0.977 — confirming the help-where-asymmetry-is-high behavior (the magnitude is capped by the easy subsample; a larger `--benchmark-distractors` pool would amplify it). Wiring proven independently (role-aware deterministic-embedder test + Ollama probe showing nomic produces distinct vectors for raw / `search_query:` / `search_document:`). *(Audit §1.2)*
- [x] **H08** · P1 · S · retrieval — **RRF weights hardcoded + asymmetric-k formula unreviewable.** Move k to config (or derive from Mode), expose via flag; document the asymmetry as intentional OR collapse to single k=60. ✅ COMPLETE (spec 009): collapsed the asymmetric per-list `kVec=40`/`kFTS=60` into ONE symmetric `k` (default 60) using the standard `score(d)=Σ 1/(k+rank)`; configurable via `rrf_k` (config) + `--rrf-k` / REST / gRPC (proto field 6) / MCP — all four transports; effective k = `req>0 ? req : cfg.EffectiveRRFK() : 60`; non-hybrid mode is a no-op; formula pinned by a deterministic unit test (2/61, 1/62) + a cross-transport parity test. Gates: build/vet/test green (18 pkgs); eval PASS (recall@10 1.000 — collapse recall-neutral, no re-baseline needed). *(Audit §1.3/§1.4)*
- [x] **H12** · P1 · S · embeddings — **Whole-doc embed batch unbounded → OOM/timeout.** Batch texts ~32–64 inside `Ollama.Embed`, concatenate responses, per-batch retry. ✅ COMPLETE (spec 010): `Ollama.Embed` batches texts at `embedBatchSize=32`, retrying each batch independently (3× backoff, 5xx-retry/4xx-fail-fast, ctx-respecting) and concatenating in order; ctx honoured between batches; a permanently-failed batch fails the whole call with NO partial result (FR-006 — structurally free, `processJob` already fails the doc on any `Embed` error). `Embedder` interface + pipeline caller + query path unchanged (Principle V). Gates: build/vet/test green (18 pkgs, +6 embed tests: bounded requests, transient retry, permanent fail no-partial, order preserved, count-mismatch rejected, edge cases). *(Audit §1.2)*

## Phase 2 — Biggest latency win (validate timing with H02)

- [x] **H01** · P0 · M · storage/latency — **Per-query full index rebuild (`LoadIndex` every Query).** Cache the loaded `(FTS, *Vector)` on the Engine with a content-hash generation counter; invalidate on ingest/delete/watcher. The single biggest latency win; also makes eval timing realistic. ✅ COMPLETE (spec 011): the Engine owns ONE shared `(*FTS,*Vector)`, seeded once via `LoadIndex` and reused by every Query (no per-query rebuild — the single biggest latency win); the pipeline/watcher/migrate mutate it live (FTS populated synchronously at chunk-store so keyword queries work immediately after the ACK; vectors land async via `processJob`); `DeleteDoc` became a `*Pipeline` method that also clears `fts`/`vec` (no phantom hits). Chose shared-live over the audit's generation-snapshot (avoids per-write full rebuild + the stale-snapshot-pin hazard). Gates: build/vet/test green (18 pkgs, +6 cache tests incl. seed-once pointer-identity + `-race` concurrency); cross-transport parity + eval PASS unchanged (FR-008). *(Audit §1.3/§1.7)*

## Phase 3 — Retrieval-quality cluster (measure each with H02)

The whole point of building H02 first: each of these can now be proven to help.

- [x] **H05** · P1 · M · retrieval — **No query transformation.** Lightweight normalization now (case/whitespace); pluggable `QueryTransformer` interface so HyDE/multi-query land behind `internal/index` without Ollama coupling. ✅ COMPLETE (spec 012): added a `QueryTransformer` interface in `internal/index` (mirrors the `Reranker` pattern — default pure, future HyDE/multi-query land in an adapter, `internal/index` stays Ollama-free) with a default `NormalizingTransformer` (Unicode case-fold + whitespace collapse/trim via `strings.Fields`; idempotent, CJK/accent-safe); wired once at the top of `Engine.Query` so the normalized query reaches the H03 guard, the H07 query-prefix embed, and both FTS+vector paths; interface returns `[]string` to future-proof multi-query (fan-out is future work). Query-only normalization (docs stay verbatim-embedded); the minor vector case-asymmetry is gated by SC-002. Gates: build/vet/test green (18 pkgs, +4 tests: normalization, idempotent, Unicode, empty→error, custom-transformer-honored); `make test-eval` PASS (recall@10 1.000 — harness queries are clean so normalization is a no-op there, gate catches accidental breakage); cross-transport parity unchanged. *(Audit §1.4)*
- [x] **H10** · P1 · M/S · chunking — **No boundary-aware chunking + doc comment lies about it.** Implement the documented paragraph→sentence→word cascade (M) OR correct the misleading package doc (S). Decide which. ✅ COMPLETE (spec 013, clarify Q1 → Option A: implement the cascade): `Splitter.Split` is now a boundary-aware **paragraph→sentence→word cascade** — greedy-fills to the Size budget, snaps chunk ends to sentence boundaries, hard-flushes at paragraph boundaries, and word-splits an over-long sentence (word fallback). **Greedy-fill refinement (Option B):** a document (or trailing section) that fits one Size budget stays a single coherent chunk — boundaries are respected *when a split is forced* (the audit's "don't cut mid-paragraph when you must cut," not "force every paragraph into its own chunk"); this keeps small docs whole (better BM25) and is backward-compatible. Pure-Go sentence/paragraph detection (ASCII `.!?` + CJK `。！？`, blank-line paragraphs; no NLP). Two overlap bugs found & fixed during impl (re-slicing small chunks; stepping backward across paragraphs); heading-fragments merge into their section. Package doc corrected to match. Gates: build/vet/test green (18 pkgs, +5 boundary tests); `make test-eval` PASS — recall@10 1.000 **unchanged** (the cascade reproduces the original chunks for the small golden docs, so no re-baseline, no regression). *(Audit §1.1)*
- [x] **H14** · P1 · M · retrieval/storage — **No metadata filtering at retrieval.** Optional `Filter` (source/type/tags) in `Search` — pre-FTS filter plus post-filter on vector hits. ✅ COMPLETE (spec 014): `index.Filter{Source,Type,Tags}` with glob/prefix source matching, exact case-insensitive type, conjunction tags from `Metadata["tags"]`; applied as a `keep(chunkID)` predicate set on the `Retrieval` struct (via `SetFilter`, mirroring `SetRRFK`) — filters FTS + Vector candidate lists **pre-fusion** (non-matching chunks never reach RRF/collapse/rerank); engine builds `keep` from `QueryRequest.Filter` + the existing `lookupChunk`/`lookupDoc` resolvers; `engine.NewFilter(...)` helper so transports don't import index directly. Exposed on all 4 transports: CLI (`--source` wired + `--type`/`--tags`), REST (JSON fields), gRPC (proto fields 7-9, regen'd), MCP (schema + renderQuery). Opt-in: nil/empty filter = byte-identical to today. Gates: build/vet/test green (18 pkgs, +3 filter tests: Matches unit, pre-fusion structural, engine source-scoping); eval PASS (recall@10 1.000, no regression). *(Audit §1.3/§1.4)*
- [x] **H15** · P1 · M · retrieval/chunking — **No parent-child / context expansion (plumbing exists, unused).** `ContextWindow` option that fetches sibling chunks via the existing `PreviousChunkID`/`NextChunkID`. ✅ COMPLETE (spec 015): populated the **previously-empty** `Chunk.PreviousChunkID`/`NextChunkID` linked list in `pipeline.processFile` (verified unpopulated → 3-line populate loop); added `QueryRequest.ContextWindow` (int, default 0 = off) + `ContextChunk{ChunkID,Content,Direction}` on `QueryHit.Context` (nil when off); engine `expandContext` follows the linked list N steps each way (previous chain reversed for document order, next chain forward), fetching sibling text via `lookupChunk` — purely additive (post-ranking, doesn't affect top-k). Exposed on all 4 transports: CLI `--context-window`, REST `context_window`, gRPC proto `context_window=10` (regen'd), MCP schema + renderQuery. Boundary chunks handled gracefully (only available siblings). Gates: build/vet/test green (18 pkgs, +3 context tests: expansion + zero-no-context + top-k-unchanged); eval PASS (recall@10 1.000 — default ContextWindow=0, no regression). Existing vaults need re-ingestion (Reprocess) to populate the linked list; old chunks return no context gracefully. *(Audit §1.4)*

## Phase 4 — Perf + robustness

- [x] **H06** · P1 · M · latency/embed — **No caching (query-result + query-embedding).** LRU keyed on `(query,mode,k,gen)` for results and `(model,query)` for embeddings; flush on `Migrate`. *(Audit §1.7/§1.8)* ✅ COMPLETE (spec 016): two in-process, bounded, exact-match LRUs in `internal/engine` — a **result cache** (full query shape + an engine-owned atomic **index epoch** → `*QueryResult`) checked at the top of `Engine.Query`, and a **query-embedding cache** (embedding profile + prefixed query → vector) wrapping the query-embed closure. Both hand-rolled on stdlib (`container/list`+`sync`) — **no new dependency** (Principle III). Invalidated by the index epoch, which bumps at **every** shared-index mutation including the **asynchronous `processJob` vector-add** (the case a write-ACK-only bump would miss — the central correctness gate, proven by a regression test that asserts the epoch advances +2/doc); `Migrate` flushes both. Default-on (transparent — a hit is byte-identical to cold), with a `nocache` override on all four transports (CLI `--no-cache`, REST/gRPC `no_cache` proto field 11, MCP) + a global kill-switch (`query_cache_enabled`) + per-cache capacities (`query_cache_results`/`query_cache_embeddings`, default 256/512); `status` surfaces `cache: result …, embedding …` (hit/miss/size/capacity). Backward-compat: an old `config.json` (pre-016) keeps caching on (absent key = default). **Found in validation:** the MCP server minted a fresh engine per tool dispatch, so MCP queries never shared a cache and `go_rag_status` read an empty cache — fixed by sharing the daemon's single engine across MCP/REST/gRPC (`NewWithEngine`), which also lets MCP share H01's seeded index. Cache is in-process/daemon-only (one-shot `go-rag query` starts cold — documented). Gates: build/vet green (`-race -cover`, engine 79.1%); `make test-eval` PASS (recall@10 1.000, eval engine cold → no quality regression, SC-006). `golangci-lint` skipped — not installed in env. *(Audit §1.7/§1.8)*
