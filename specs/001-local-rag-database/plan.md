## Summary

Build go-rag v1: a single-binary, local RAG database. Users set up, ingest a folder
of documents (PDF/Word/Markdown/text), and query it with grounded, source-cited
answers. Technical approach (from the PRD): a pure-Go, CGo-free binary using cobra
for the CLI, an embedded Pebble LSM store for all state, chromem-go for vector
search, pdfcpu for PDF extraction, fsnotify+polling for change detection, and a
local Ollama instance for embeddings. Writes acknowledge in <10ms; all indexing is
asynchronous after the ack.

## Technical Context

**Language/Version**: Go 1.26 (PRD requires 1.22+; toolchain installed is 1.26.4).

**Primary Dependencies**:
- `github.com/spf13/cobra` — CLI framework (the six commands)
- `github.com/cockroachdb/pebble` — embedded LSM-tree KV store (all state)
- `github.com/philippgille/chromem-go` — embedded HNSW vector store
- `github.com/pdfcpu/pdfcpu` — PDF text + metadata extraction
- `github.com/fsnotify/fsnotify` — cross-platform file watching
- Embeddings: Ollama `/api/embed` HTTP endpoint (no Go library; abstracted behind `Embedder`)

**Storage**: One embedded Pebble instance, single-byte-prefix key-space (PRD §6.7).
Vectors held in chromem-go (in-memory + disk persistence). No external DB.

**Testing**: `go test` (stdlib), table-driven; integration tests exercise real Ollama
+ real PDFs/fixtures; benchmark tests guard the <10ms write and <500ms query budgets.

**Target Platform**: Linux (primary), macOS, Windows. Static binary, user-space, no root.

**Project Type**: CLI tool (single binary), also exposed as MCP tools (PRD G7).

**Performance Goals** (PRD §10.1): write ACK <10ms; hybrid query <500ms (top-5);
keyword query <50ms; vector search <100ms (top-60); cold start <1s; >10 chunks/sec
sustained ingest throughput.

**Constraints**: pure Go (`CGO_ENABLED=0`); single-writer process; local-only; SHA-256
idempotent ingestion; <10ms write-ACK budget independent of corpus size; binary <25MB;
idle memory <50MB.

**Scale/Scope**: 100 → 100K documents (v1 target); 512-token chunks, 50-token overlap.

scaffold already exists from project setup; this feature implements the bodies behind
the existing interfaces/stubs.
